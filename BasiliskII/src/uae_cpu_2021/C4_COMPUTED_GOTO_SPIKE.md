# C.4 Computed-Goto Dispatch Spike

## Scope

Introduce a computed-goto (threaded interpreter) dispatch path for the 68k interpreter in `uae_cpu_2021`, behind a build-time flag, while preserving existing behavior as the default path.

## Current Baseline

- Dispatch today is indirect-call based in `newcpu.cpp`:
  - `(*cpufunctbl[opcode])(opcode)`
- Opcode handlers and function tables are generated by `gencpu.c`.
- Unix and macOS build flows generate `cpuemu.cpp`, `cpustbl.cpp`, and `cpufunctbl.cpp` from generator tools.

## Milestones

### M1 — Dispatch seam (safe, no behavior change)

- Isolate opcode dispatch behind dedicated helper entry points in `newcpu.cpp`.
- Keep default path functionally identical.
- Add compile-time flag plumbing (`ENABLE_THREADED_DISPATCH`) with fallback to indirect-call dispatch.

### M2 — Generator output for threaded labels

- Extend `gencpu.c` to optionally emit a threaded-dispatch include fragment with per-opcode labels and jump targets.
- Keep generated artifacts deterministic and reproducible.

Current scaffold artifact:

- `threaded_dispatch_map.inc` generated by `gencpu.c` (opcode -> label -> resolved handler mapping via `THREADED_DISPATCH_MAP_ENTRY(opcode, label, handler)`)

### M3 — Interpreter threaded loop

- Add GCC/Clang-only computed-goto loop in `newcpu.cpp`:
  - fetch opcode
  - jump to label target
  - execute generated opcode body
  - check ticks / spcflags / specialties
  - loop
- Preserve exception and fault-PC behavior.

Current scaffold status:

- `newcpu.cpp` now contains an `ENABLE_THREADED_DISPATCH` + GCC/Clang guarded computed-goto loop.
- If `threaded_dispatch_map.inc` is unavailable, code falls back to the classic indirect function-table dispatch.

### M4 — Validation and fallback hardening

- Build and run with threaded dispatch disabled/enabled.
- Validate correctness paths (exceptions, STOP, interrupts, MMU paths if enabled).
- Keep non-GNU toolchains and unsupported targets on existing indirect-call path.

## Risks

- Generator complexity and merge surface with existing generated files.
- Subtle control-flow regressions around exception exits and specialty checks.
- Toolchain variance for labels-as-values extension.

## Success criteria

- Default build remains unchanged and stable.
- Optional threaded-dispatch build compiles and runs identical test workloads.
- Measurable interpreter speedup on ARM/AArch64 workloads.

## Next immediate coding step

Finish M4 by validating a full interpreter build with `--enable-threaded-dispatch` in an environment with required FPU dependencies available.

## Validation command blocks

Use two command blocks for full coverage. A third block is optional for quick reruns.

### Block 1 — Configure + build + compile validation (required)

Run from repository root:

```bash
pushd BasiliskII/src/Unix >/dev/null
./configure \
  --enable-threaded-dispatch \
  --disable-jit-compiler \
  --enable-fpe=uae \
  --with-gtk=no --with-mon=no --with-esd=no \
  --with-bincue=no --with-libvhd=no --with-vdeplug=no --with-vncserver=no
mkdir -p obj
make -j2 obj/gencpu
./obj/gencpu
make -j2 obj/newcpu.o
popd >/dev/null
```

### Block 2 — Cleanup generated artifacts (required)

Run from repository root:

```bash
rm -f BasiliskII/src/Unix/threaded_dispatch_map.inc
```
